阶段升级说明


您在进行升级之前请仔细阅读《升级注意事项》。
阅读完毕请按照以下步骤进行升级：

从当前版本升级到 SiteServer 系列产品3.5 版本
（SqlServer、Access及Oracle数据库升级方式一致）

    第 1 步：将系统程序更新上传到原系统目录（注意：不要将程序目录下的SiteFiles子目录删除，否则站点相关文件会丢失）

    ./Bin（请将原目录删除，重新上传）
    ./SiteServer（请将原目录删除，重新上传）
    ./UserCenter（请将原目录删除，重新上传）
    ./SiteFiles/bairong（请将原目录删除，重新上传）
    ./SiteFiles/Configuration（请将原目录删除，重新上传）
    ./SiteFiles/Inner（请将原目录删除，重新上传）
    ./SiteFiles/Module（请将原目录删除，重新上传）
    ./SiteFiles/SSO（上传）

    第 2 步：手动配置 web.config 文件

    （如果是从3.4.3版本升级无需此步骤）

    用文本编辑器打开系统根目录下的web.config文件，找到
    <httpModules>
    <add name="SiteServer" type="SiteServer.Provider.HttpModule.SiteServerHttpModule, SiteServer.Provider" />
    </httpModules>
    <httpHandlers>
    <add verb="GET" path="*/SiteServer_Redirect.aspx" type="SiteServer.Provider.HttpHandler.Redirect, SiteServer.Provider" />
    <add verb="GET" path="*/UserUtils.aspx" type="SiteServer.Provider.HttpHandler.UserUtils, SiteServer.Provider" />
    <add verb="GET" path="*/SiteServer_AjaxUtils.aspx" type="SiteServer.Provider.HttpHandler.AjaxUtils, SiteServer.Provider" />
    <add verb="GET,POST" path="*/*_v.aspx" type="SiteServer.Provider.HttpHandler.View, SiteServer.Provider" />
    </httpHandlers>

    替换为：

    <httpModules>
    <add name="SiteServer" type="BaiRong.Provider.HttpModule.SiteServerHttpModule, BaiRong.Provider" />
    </httpModules>

    第 3 步：运行升级程序

    请在浏览器访问 http://您的域名/siteserver/upgrade/index.aspx ，然后按照提示操作即可。

    第 4 步：手工把以前建的内容辅助表增加以下字段

    ConsumePoint int DEFAULT 0 NOT NULL

    第 5 步：把SiteFiles\Configuration\Configuration.config的数据库连接类型和连接字符串改为和web.config一致

    至此，升级完毕。

升级方式二：

  • 将系统内各网站保存为站点模板（站点模板保存在SiteFiles/SiteTemplates目录下）
  • 重新安装SiteServer 系列产品3.5
  • 将保存的站点模板拷贝到新版本的SiteFiles/SiteTemplates目录下
  • 选择对应的站点模板创建网站，将站点模板导入新系统
  • 升级完毕
